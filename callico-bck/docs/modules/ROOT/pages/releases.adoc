[#0-6-5]
== 0.6.5

Released on *26 February 2025* &bull; View on https://gitlab.teklia.com/callico/callico/-/releases/0.6.5[GitLab]

* More precise IIIF ratio algorithm in elements thumbnails, using the `max` image size feature instead of `full`
* Upgrade Django to 5.1
* Increase `IMAGE_QUALITY` constant in `InteractiveImage` to provide a deeper zoom level for large images
* Allow users to set the entity colors in entity type campaigns
* Support full word selection on entity campaigns
* During Arkindex export, do not log an error when an `Elements` or `ElementGroup` annotation is empty
* Remove the beta mode `Transcription on the image` from `Entity` campaigns

[#0-6-4]
== 0.6.4

Released on *31 May 2024* &bull; View on https://gitlab.teklia.com/callico/callico/-/releases/0.6.4[GitLab]

* The IIIF image and the element thumbnail URLs are included in new columns in CSV and XLSX exports.
* Initial annotations for *Elements* campaigns are now pre-filled with imported elements, if they match xref:./campaigns/configure/elements.adoc#configuration-parameters[the configured *Element types to use to annotate*]. This allows users to validate or correct Machine Learning predictions.

image:releases/064-pre-filled-segmentation.png[Pre-filled segmentation task]

* As for *Entity Form* campaigns, to facilitate the validation of imported transcriptions during *Transcription* annotations, there is a new configuration option to define a confidence threshold below which the imported data should be corrected by contributors. Such data is highlighted in red during annotation.
* Thumbnail height has been restricted across the application to prevent very tall images from breaking Callico's display.
* To cleanup a project, managers can xref:./campaigns/close_reopen_archive.adoc#archive-a-campaign[archive campaigns] from their details page, making them completely inaccessible.
* Project creation, a feature originally available only to staff users, can now be enabled for all users by adding the `PROJECT_CREATION_ALLOWED` Django setting to the instance.
* *Entity Form* campaign fields can be configured using a list of authority values, such fields will be displayed as `<input/>` elements, linked to a filterable dropdown listing allowed annotations. Authority records can be ingested by an instance administrator using a new Django command.
* *_For developers:_* Various linting tools (`flake8`, `isort` and `black`) have been replaced by https://docs.astral.sh/ruff/[Ruff].

[discrete#django-commands-064]
==== Django commands

* `ingest-authority` allows to ingest authority values (along with an optional identifier and metadata) from a CSV file.

[#0-6-3]
== 0.6.3

Released on *25 March 2024* &bull; View on https://gitlab.teklia.com/callico/callico/-/releases/0.6.3[GitLab]

* The number of comments is included in a new column in CSV and XLSX exports.

[#0-1-1]
== 0.1.1

Released on *27 April 2022* &bull; View on https://gitlab.teklia.com/callico/callico/-/releases/0.1.1[GitLab]

* Allow contributors to:
 ** annotate *Transcription* tasks,
 ** skip an annotation,
 ** list their campaigns,
 ** list their assigned tasks.
* *Transcription* tasks now support the annotation of filtered child elements in parallel to the assigned one.

[discrete#django-commands-011]
==== Django commands

* Improve the command to import Arkindex elements, `import-arkindex-elements`, to support specifying a parent folder to import.

[#0-1-0]
== 0.1.0

Released on *14 April 2022* &bull; View on https://gitlab.teklia.com/callico/callico/-/releases/0.1.0[GitLab]

Initial release, allowing to create a project, import Arkindex elements, and assign simple xref:./campaigns/configure/transcription.adoc[text *Transcription* tasks] to users.

Managers can also list projects, elements, campaigns, tasks and unassign pending tasks.
